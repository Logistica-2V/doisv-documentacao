Recurso: Autenticação

Todos os endpoints nesta seção são públicos e não exigem autenticação.

1. Login de Lojista

Autentica um lojista no sistema e retorna um token de acesso.

POST /doisv/login

Request Body:

{
  "email": "lojista@email.com",
  "password": "sua_senha_secreta"
}

Response Body:

{
  "token": "string"
}

Status Codes:

  • 200 OK: Autenticação bem-sucedida.
  • 401 Unauthorized: Credenciais inválidas.
  • 422 Unprocessable Entity: Erro de validação.

2. Login de Consumidor

Autentica um consumidor no sistema usando o serial da venda e a senha, retornando um token de acesso.

POST /doisv/login/consumidores

Request Body:

{
  "serial": "VENDA-20251017-XYZ",
  "senha": "sua_senha_secreta"
}

Response Body:

{
  "token": "string"
}

Status Codes:

  • 200 OK: Autenticação bem-sucedida.
  • 401 Unauthorized: Serial ou senha inválidos.
  • 422 Unprocessable Entity: Erro de validação.

3. Recuperar Senha do Lojista

Envia um código de recuperação de senha para o e-mail do lojista.

POST /doisv/login/lojista/recuperar-senha

Request Body:

{
  "email": "lojista@email.com"
}

Response Body:

"Código de recuperação enviado para o email lojista@email.com"

Status Codes:

  • 200 OK: E-mail de recuperação enviado com sucesso.
  • 404 Not Found: Lojista não encontrado com este e-mail.
  • 422 Unprocessable Entity: Usuário inativo.

4. Validar Código de Recuperação

Valida o código de recuperação de senha enviado por e-mail para o lojista.

POST /doisv/login/lojista/validar-recuperacao

Request Body:

{
  "email": "lojista@email.com",
  "codigoRecuperacao": "123456"
}

Response Body:

Sem corpo de resposta (resposta vazia).

Status Codes:

  • 200 OK: Código de recuperação válido.
  • 404 Not Found: Código de recuperação inválido.
  • 410 Gone: Código de recuperação expirado.
  • 422 Unprocessable Entity: Código já utilizado.

5. Atualizar Senha do Lojista

Atualiza a senha do lojista após a validação do código de recuperação. A nova senha deve ter no mínimo 8 caracteres.

POST /doisv/login/lojista/atualizar-senha

Request Body:

{
  "email": "lojista@email.com",
  "codigoRecuperacao": "123456",
  "novaSenha": "nova_senha_forte"
}

Response Body:

"Senha atualizada com sucesso."

Status Codes:

  • 200 OK: Senha atualizada com sucesso.
  • 404 Not Found: Código de recuperação inválido.
  • 410 Gone: Código de recuperação expirado.
  • 422 Unprocessable Entity: Senha fraca (mínimo 8 caracteres) ou código já utilizado.

Recurso: Lojas

Todos os endpoints nesta seção exigem autenticação via Bearer Token e são restritos ao perfil Lojista.

1. Buscar Todas as Lojas

Retorna uma lista com todas as lojas cadastradas.

GET /doisv/lojas

Response Body:

[
  {
    "idLoja": 1,
    "nome": "Loja de Calçados Central",
    "cnpj": "11.111.111/0001-11",
    "segmento": "Calçados",
    "logo": "http://example.com/logo1.png",
    "email": "contato@calcadoscentral.com",
    "status": "ATIVO"
  }
]

Status Codes:

  • 200 OK: Requisição bem-sucedida.
  • 401 Unauthorized: Token inválido.

2. Buscar Loja por ID

Retorna os detalhes de uma loja específica.

GET /doisv/lojas/{id}

Response Body:

{
  "idLoja": 1,
  "nome": "Loja de Calçados Central",
  "cnpj": "11.111.111/0001-11",
  "segmento": "Calçados",
  "logo": "http://example.com/logo1.png",
  "email": "contato@calcadoscentral.com",
  "status": "ATIVO"
}

Status Codes:

  • 200 OK: Requisição bem-sucedida.
  • 401 Unauthorized: Token inválido.
  • 404 Not Found: Loja não encontrada.

3. Criar uma Nova Loja

Cria um novo registro de loja. A requisição deve ser do tipo `multipart/form-data`.

POST /doisv/lojas

Request Body (multipart/form-data):

Parte 1: `loja` (application/json)

{
  "nome": "Nova Loja de Artigos Esportivos",
  "cnpj": "33.333.333/0001-33",
  "segmento": "Esportes",
  "email": "contato@esportes.com"
}

Parte 2: `logo` (image/png, image/jpeg, etc.) - Arquivo da imagem

Response Body:

{
  "idLoja": 3,
  "nome": "Nova Loja de Artigos Esportivos",
  "cnpj": "33.333.333/0001-33",
  "segmento": "Esportes",
  "logo": "http://example.com/novo_logo.png",
  "email": "contato@esportes.com",
  "status": "ATIVO"
}

Status Codes:

  • 201 Created: Loja criada com sucesso.
  • 400 Bad Request: Requisição malformada.
  • 401 Unauthorized: Token inválido.
  • 409 Conflict: CNPJ já cadastrado.
  • 422 Unprocessable Entity: Erro de validação.

4. Atualizar uma Loja

Atualiza os dados de uma loja existente. Requisição `multipart/form-data`.

PUT /doisv/lojas/{id}

Request Body (multipart/form-data):

Parte 1: `loja` (application/json)

{
  "nome": "Loja de Esportes Radical",
  "segmento": "Esportes Radicais"
}

Parte 2: `logo` (image/png, image/jpeg, etc.) - Novo arquivo da imagem

Response Body:

{
  "idLoja": 3,
  "nome": "Loja de Esportes Radical",
  "cnpj": "33.333.333/0001-33",
  "segmento": "Esportes Radicais",
  "logo": "http://example.com/logo_atualizado.png",
  "email": "contato@esportes.com",
  "status": "ATIVO"
}

Status Codes:

  • 200 OK: Loja atualizada com sucesso.
  • 401 Unauthorized: Token inválido.
  • 403 Forbidden: Acesso negado.
  • 404 Not Found: Loja não encontrada.
  • 422 Unprocessable Entity: Erro de validação.

5. Desativar uma Loja (Soft Delete)

Desativa uma loja, mantendo o registro no banco de dados.

PATCH /doisv/lojas/{id}

Status Codes:

  • 204 No Content: Loja desativada com sucesso.
  • 401 Unauthorized: Token inválido.
  • 403 Forbidden: Acesso negado.
  • 404 Not Found: Loja não encontrada.

6. Deletar uma Loja (Permanent Delete)

Remove permanentemente o registro de uma loja do sistema.

DELETE /doisv/lojas/{id}

Status Codes:

  • 204 No Content: Loja removida com sucesso.
  • 401 Unauthorized: Token inválido.
  • 403 Forbidden: Acesso negado.
  • 404 Not Found: Loja não encontrada.

POST /doisv/produtos

Request Body (multipart/form-data):

Parte 1: produto (application/json)

{
  "descricao": "Camiseta Branca",
  "unidadeMedida": "UN",
  "preco": 94.90
}

Parte 2: imagem (image/*) - Imagem do produto

Response Body:

{
  "idProduto": 1,
  "descricao": "Camiseta Branca",
  "unidadeMedida": "UN",
  "preco": 94.90,
  "imagem": "1a2b3c4d5e",
  "status": "ATIVO"
}

Status Codes:

  • 201 Created: Produto criado com sucesso.
  • 400 Bad Request: Tipo de arquivo de imagem inválido.
  • 401 Unauthorized: Token inválido.
  • 403 Forbidden: Erro de permissão ao acessar o Google Drive.
  • 409 Conflict: Já existe um produto com a mesma descrição nesta loja.
  • 422 Unprocessable Entity: Erro de validação nos campos.

Os endpoints 1 e 3 exigem autenticação via Bearer Token (lojista). O endpoint 2 (métricas públicas) não requer autenticação.

1. Métricas Privadas da Loja

Retorna métricas detalhadas da loja do lojista autenticado, incluindo notas médias por tipo de feedback, quantidade de feedbacks, percentual de solicitações por tipo (troca/devolução) e as avaliações resumidas. O parâmetro periodo define o número de dias retroativos (padrão: 365). Acesso: Lojista

GET /doisv/metricas/privadas?periodo=365

Parâmetros:

  • periodo (Integer, query, opcional) - Número de dias retroativos. Padrão: 365.

Response Body:

    "loja": 4.50,
    "solicitacao": 4.20

  "totalVendas": 15,
  "quantidadeFeedbackPorTipo": {

    "solicitacao": 7
  },
  "percentualPorTipo": {
    "troca": 66.67,
    "devolucao": 33.33
  },
  "avaliacoes": {
    "loja": [
      {
        "idSolicitacao": 1,
        "nomeConsumidor": "Ana",
        "comentario": "Ótimo atendimento!",
        "nota": 5,
        "dataFeedback": "25/10/2025"
      }
    ],
    "solicitacao": [
      {
        "idSolicitacao": 2,

Status Codes:

  • 200 OK: Requisição bem-sucedida.
  • 401 Unauthorized: Token inválido ou não fornecido.
  • 422 Unprocessable Entity: Nenhum feedback encontrado para o período ou parâmetros inválidos.

2. Métricas Públicas de Todas as Lojas

Retorna uma lista paginada com métricas públicas de todas as lojas, incluindo notas médias e total de avaliações. Acesso: Público

GET /doisv/metricas/publicas?periodo=180&page=0&size=10

Parâmetros:

3. Quantidade de Solicitações por Status

Retorna a contagem de solicitações agrupadas por status para a loja do lojista autenticado. Todos os status possíveis são retornados, mesmo que com valor zero. Acesso: Lojista

GET /doisv/metricas/solicitacoes/por-status?periodo=365

Parâmetros:

  • periodo (Integer, query, opcional) - Número de dias retroativos. Padrão: 365.

Response Body:

{
  "pendente": 3,
  "aprovada": 5,
  "rejeitada": 1,
  "cancelada": 2,
  "em_andamento": 1,
  "em_transito": 0,
  "concluida": 4
}

Status Codes:

  • 200 OK: Requisição bem-sucedida.
  • 401 Unauthorized: Token inválido ou não fornecido.
  • 422 Unprocessable Entity: Parâmetros inválidos (período nulo ou negativo).
